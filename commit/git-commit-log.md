### fix: 修复测试点索引计算错误 - 使用与显示相同的排序逻辑

### fix: 修复草稿箱排序问题和同场景去重问题

**问题描述：**
1. **排序问题**：1.2.1 显示在 1.8.1 后面，序号顺序不对
2. **去重问题**：1.6.1 和 1.6.2 用例名称完全相同（"用户名输入超长字符"），但没有被去重

**问题分析：**

1. **排序问题的原因**
   - 之前按测试点在 `testScenarios.testPoints` 数组中的索引排序
   - 但如果测试点是乱序添加的，索引就不能反映正确的序号
   - 需要从测试点名称中提取数字序号进行排序

2. **去重问题的原因**
   - 之前的去重逻辑跳过了"同一场景"的所有用例
   - 导致同一场景内不同测试点的重复用例无法去重
   - 例如：1.6.1 和 1.6.2 都是"用户名输入超长字符"，但因为在同一场景而没有去重

**修复方案：**

1. **修复排序逻辑**（FunctionalTestCaseGenerator.tsx）
   - 新增 `extractTestPointNumber` 函数，从测试点名称中提取数字序号
   - 例如："1.2-用户名为空" -> 提取 1.2 -> 转换为 1002（major * 1000 + minor）
   - 排序时按提取的数字序号排序，而不是数组索引
   - 没有序号的测试点排在最后

2. **修复去重逻辑**（FunctionalTestCaseGenerator.tsx）
   - 改为支持同一场景内不同测试点之间的去重
   - 只跳过"同一测试点"的用例（这些已经在重新生成时被删除）
   - 同一场景内不同测试点的用例如果名称相同，会被去重
   - 在比较名称时，移除序号前缀（1.2.3- 或 1.2-）后再比较

**去重示例：**

✅ 会去重：
```
场景1 - 测试点6.1："1.6.1-用户名输入超长字符"
场景1 - 测试点6.2："1.6.2-用户名输入超长字符"  // 移除序号后名称相同 → 去重 ✅

场景1 - 测试点1："用户名为空"
场景2 - 测试点3："用户名为空"  // 跨场景名称相同 → 去重 ✅
```

❌ 不会去重：
```
场景1 - 测试点1："用户名为空"
场景1 - 测试点1："用户名为空"  // 同一测试点（重新生成时已删除旧的）

场景1 - 测试点1："用户名为空"
场景1 - 测试点2："密码为空"  // 名称不同 → 不去重
```

**修复效果：**
- ✅ 草稿箱按正确的数字序号排序（1.1, 1.2, 1.3, ..., 1.10, 1.11）
- ✅ 同一场景内不同测试点的重复用例会被去重
- ✅ 跨场景的重复用例也会被去重
- ✅ 只有同一测试点的用例不会互相去重（因为重新生成时已删除）

**影响文件：**
- `src/pages/FunctionalTestCaseGenerator.tsx`
