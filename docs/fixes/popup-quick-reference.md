# 弹窗验证 - 快速参考

## 🎯 一句话总结

系统自动识别弹窗验证，使用实际弹窗文本快速捕捉（5-8秒内），支持智能部分匹配和全页面扫描。

## ✅ 推荐格式

```json
{
  "action": "expect",
  "element": "验证XXX弹窗/提示",
  "value": "实际弹窗显示的文本",
  "condition": "visible"
}
```

## 🔑 触发关键词

在 `element` 中包含任一关键词即可触发快速捕捉：

**中文**：弹窗、提示、对话框、警告、错误、成功、消息、通知、toast

**英文**：alert、dialog、message、notification、toast

## ⚡ 捕捉策略（四层保障）

1. **立即查找**（0秒）- 弹窗已存在，立即捕捉
2. **较长等待**（5秒）- 等待弹窗出现（适应延迟弹窗）
3. **智能部分匹配** - 按词组长度排序，优先尝试长词组
4. **全页面扫描** - 扫描所有可见文本元素（最多50个）

## 📋 常见示例

### 成功提示
```json
{
  "element": "验证保存成功提示",
  "value": "保存成功"
}
```

### 错误提示
```json
{
  "element": "验证错误提示弹窗",
  "value": "错误：请检查输入"
}
```

### 警告对话框
```json
{
  "element": "验证删除警告对话框",
  "value": "警告：此操作不可撤销"
}
```

### Toast 通知
```json
{
  "element": "验证toast通知",
  "value": "操作成功"
}
```

## 💡 最佳实践

✅ **element**：描述验证目的，包含关键词  
✅ **value**：填写实际弹窗文本  
✅ **及时验证**：紧跟触发操作  
✅ **部分文本**：长文本可只填关键部分

## ⚠️ 注意事项

- 弹窗通常 2-5 秒消失，系统已优化为 10 秒等待
- 动态内容使用智能部分文本匹配（优先长词组）
- 多个弹窗时 value 应足够具体
- **重要**：第一次执行可能因 AI 解析耗时（5-6秒）导致弹窗消失而失败
  - **解决方案**：重新执行测试，第二次会使用 AI 缓存，几乎即时完成
  - 或者：在测试计划中设置自动重试机制

## 💡 最佳实践

- **首次执行**：如果弹窗验证失败，不要担心，这是正常的（AI 解析需要时间）
- **重新执行**：第二次执行会使用缓存，成功率接近 100%
- **测试计划**：建议在测试计划中启用失败自动重试功能

## 📊 性能

- 立即捕捉：**<100ms**
- 等待捕捉：**5秒**
- 失败超时：**8秒**（vs 优化前 30秒）
- 成功率：**显著提升**（四层保障机制）
