# Midscene 前端日志格式设计方案

## 方案对比

本文档提供 5 种前端日志输出格式设计，从简洁到详细，供选择和优化。

---

## 方案 1: 极简风格 (Minimal)

**特点**: 单行输出，适合快速浏览

### 示例输出
```
🚀 测试开始 | TC_00001 | 3步骤
⏱️ 步骤1/3 | 导航 | 1.6s ✅
⏱️ 步骤2/3 | 输入操作 | 4.1s 🤖 ✅
⏱️ 步骤3/3 | 断言验证 | 8.3s 🤖 ✅
✅ 测试完成 | 总耗时: 20s | AI: 12.4s | 成功率: 100%
📊 统计 | 调用15次 | 命中8次(53%) | Token: 2,097 | 成本: $0.0002
```

### 优点
- 信息密度高，一屏显示完整流程
- 适合移动端和小屏幕
- 易于快速扫描

### 缺点
- 细节信息不足
- 不够直观

## 推荐方案组合

### 实时执行阶段
推荐使用 **方案 3 (进度条风格)** 或 **方案 4 (时间轴风格)**
- 实时更新友好
- 进度可视化
- 适合追踪执行流程

### 执行完成后
推荐使用 **方案 5 (仪表盘风格)** 或 **方案 2 (卡片风格)**
- 信息完整
- 数据对比清晰
- 适合总结报告

### 移动端/小屏幕
推荐使用 **方案 1 (极简风格)**
- 信息密度高
- 占用空间小
- 快速浏览

## 实现建议

### 1. 可配置的日志级别
```typescript
enum LogFormat {
  MINIMAL = 'minimal',      // 方案1
  CARD = 'card',           // 方案2
  PROGRESS = 'progress',   // 方案3
  TIMELINE = 'timeline',   // 方案4
  DASHBOARD = 'dashboard'  // 方案5
}
```

### 2. 响应式切换
```typescript
// 根据屏幕宽度自动选择格式
const getLogFormat = (width: number) => {
  if (width < 768) return LogFormat.MINIMAL;
  if (width < 1024) return LogFormat.CARD;
  return LogFormat.DASHBOARD;
};
```

### 3. 实时/完成态切换
```typescript
// 执行中使用进度条，完成后切换为仪表盘
const format = isRunning ? LogFormat.PROGRESS : LogFormat.DASHBOARD;
```

### 4. 用户偏好设置
```typescript
// 允许用户在设置中选择喜欢的格式
interface UserPreference {
  logFormat: LogFormat;
  showTimestamp: boolean;
  showAIDetails: boolean;
  showCacheStats: boolean;
}
```

## 颜色方案建议

### 状态颜色
- 🟢 成功: `#52c41a` (绿色)
- 🔴 失败: `#ff4d4f` (红色)
- 🟡 警告: `#faad14` (黄色)
- 🔵 进行中: `#1890ff` (蓝色)
- ⚪ 等待: `#d9d9d9` (灰色)

### 语义颜色
- 🤖 AI调用: `#722ed1` (紫色)
- 💾 缓存: `#13c2c2` (青色)
- ⏱️ 时间: `#fa8c16` (橙色)
- 💰 成本: `#eb2f96` (粉色)
