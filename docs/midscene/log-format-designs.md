【⚠️ 注意事项】
✅ 可以被缓存的操作
• Plan (计划生成)
  ├─ 类型: type: plan
  ├─ 内容: 将自然语言描述转换为具体的操作步骤
  └─ 示例: "在用户名输入框输入账号: sysadmin" → 生成 workflow

• Locate (元素定位)
  ├─ 类型: type: locate
  ├─ 内容: 元素的 XPath 定位信息
  └─ 示例: "登录名输入框" → /html/body/div[1]/div[1]/div[1]/div[1]/div[3]/input[1]

❌ 不能被缓存的操作
• Assert (断言验证)
  ├─ 原因: 断言需要实时验证页面当前状态, 每次执行时页面内容可能不同
  ├─ 示例: 验证"欢迎回来"文本是否存在
  └─ 影响: 每次执行都会调用AI进行验证

• Extract (数据提取)
  ├─ 原因: 提取的数据是动态的, 每次执行时内容可能变化
  └─ 示例: 使用稳定的元素描述

• 页面内容变化的操作
  ├─ 原因: 如果页面DOM结构发生变化, 缓存的XPath可能失效
  └─ Midscene处理: 会自动检测缓存失效并更新缓存


【💡 优化建议】
• 减少断言操作
  ├─ 断言无法缓存, 每个断言都会调用AI, 产生API费用
  ├─ 建议: 优化测试用例减少不必要的断言, 减少AI调用次数
  └─ 示例: 合并多个断言, 用一个断言验证多个条件

• 稳定的页面结构
  ├─ 页面结构稳定时, locate缓存命中率更高
  ├─ 建议: 避免频繁修改页面DOM结构
  └─ 示例: 使用稳定的元素描述

• 缓存失效场景
  ├─ 页面DOM结构变化
  ├─ 元素位置或属性改变
  └─ 系统会自动检测并更新缓存
